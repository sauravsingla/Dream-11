# Security Policy

## Supported versions

This repository is maintained as research software. Security fixes are applied to the latest version on the `main` branch.

| Version | Supported |
| --- | --- |
| Latest `main` | Yes |
| Older snapshots and forks | No |

## Reporting a vulnerability

Please do not disclose a suspected vulnerability publicly before it has been reviewed.

Open a private security advisory through the repository's **Security** tab when that option is available. Include:

- a clear description of the issue;
- affected files, functions or versions;
- steps to reproduce the behaviour;
- the potential impact;
- any suggested remediation.

Please avoid including credentials, private datasets, commercial solver licence information or personally identifiable information.

## Scope

Relevant reports include vulnerabilities in the Python package, command-line workflow, dependency handling, file parsing or generated outputs. Research-model limitations, prediction quality and contest outcomes are not security vulnerabilities.

## Responsible use

This project is an academic demonstration of constrained optimisation. Users are responsible for reviewing input data, solver configuration, generated teams and applicable fantasy-sports rules before use.
